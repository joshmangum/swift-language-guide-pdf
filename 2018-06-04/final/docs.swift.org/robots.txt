    <h1>Documentation</h1>
  </header>

  <p>You can use the resources on this page as documentation for the Swift language.
Apple hosts <a href="https://developer.apple.com/swift/resources/">additional resources for learning Swift</a>—such as videos, sample code and playgrounds, and additional documentation—including 
the <a href="https://developer.apple.com/documentation/swift/swift_standard_library/">Swift standard library reference</a>.</p>

<h2 id="the-swift-programming-language">The Swift Programming Language</h2>

<p><em>The Swift Programming Language</em> is the authoritative reference for Swift,
offering a guided tour, a comprehensive guide, and a formal reference of the language.
You can read the book here:</p>

<div class="links">
  <ul>
    <li><a href="swift-book/index.html">The Swift Programming Language</a>
      <ul>
        <li><a href="swift-book/GuidedTour/GuidedTour.html">A Swift Tour</a></li>
        <li><a href="swift-book/LanguageGuide/TheBasics.html">Language Guide</a></li>
        <li><a href="swift-book/ReferenceManual/AboutTheLanguageReference.html">Language Reference</a></li>
      </ul>
    </li>
  </ul>
</div>

<p>You can also <a href="https://swift.org/documentation/TheSwiftProgrammingLanguage(Swift4.2).epub">download the latest version as an ePub</a>, available under a <a href="https://creativecommons.org/licenses/by/4.0/">Creative Commons Attribution 4.0 International (CC BY 4.0) License</a>.</p>

<h4 id="translations">Translations</h4>

<p>We encourage you to participate in translating <em>The Swift Programming Language</em> into other languages. 
Get involved with an existing translation project, or start a new one where there’s a need. 
A great example is <a href="https://github.com/numbbbbb/the-swift-programming-language-in-chinese">The Swift Programming Language (Chinese translation, GitHub project)</a>.</p>

<div class="info">
  <p>If you know of a quality translation project underway, 
please let us know on the <a href="https://forums.swift.org/c/development/compiler">Compiler Development</a> forums,
and we’ll do our best to provide a link to that project from this page.</p>
</div>

<h2 id="api-design-guidelines">API Design Guidelines</h2>

<p>A goal of the Swift 3.0 release was
to set a uniform standard for the 
programmer’s user experience, much of which is defined by the names
and idioms that appear in APIs.  These
<a href="https://swift.org/documentation/api-design-guidelines/">design guidelines</a> explain how to 
make sure that your code feels like a part of the larger Swift ecosystem.</p>

<h2 id="migration-guides">Migration Guides</h2>

<p>Xcode includes a Swift Migrator tool that helps you migrate your project to the latest version of Swift, 
or update it to work with the latest SDKs.</p>

<div class="links">
  <ul>
    <li><a href="https://swift.org/migration-guide-swift4.2/">Migrating to Swift 4.2</a></li>
    <li><a href="https://swift.org/migration-guide-swift4/">Migrating to Swift 4</a></li>
  </ul>
</div>
